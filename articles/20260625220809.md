## はじめに

ブログを継続的に更新するのは思った以上に大変です。ネタ出し、構成検討、執筆、校正…毎回このサイクルを回すのは時間的コストがかかります。

「どうせなら技術的に解決できないか？」と思い立ち、Anthropicが提供するLLM「Claude」のAPIを使ってブログ記事の自動生成パイプラインを構築してみました。本記事ではその実装過程を、コードと一緒に詳しく解説します。

なお、「自動生成 = 品質が低い」というイメージを持つ方もいると思いますが、プロンプト設計と後処理を工夫することで、実用に耐えるクオリティを実現できます。その点も含めて紹介します。

---

## 全体の仕組み（アーキテクチャ概要）

今回構築したシステムの全体像は以下のとおりです。

```
[トピック入力]
    ↓
[アウトライン生成（Claude）]
    ↓
[セクションごとの本文生成（Claude）]
    ↓
[Markdown整形・後処理]
    ↓
[ファイル出力 or CMS投稿API]
```

### なぜセクション分割して生成するのか

最初は「トピックを渡して一発で記事全体を生成する」アプローチを試しました。しかし以下の問題が発生しました。

- **トークン上限に引っかかる**: 長文を一度に出力しようとすると、途中で切れる
- **構成がバラつく**: 見出し構造が毎回異なり、品質が安定しない
- **修正しにくい**: 一部だけ再生成したいときに全体を再実行する必要がある

そこで「アウトライン（骨格）を先に確定させ、セクションごとに本文を生成する」二段階方式に切り替えました。これにより安定性と柔軟性が大幅に向上しました。

## 実装手順

### Step 1: 環境構築

まずは必要なライブラリをインストールします。

```bash
pip install anthropic python-dotenv
```

APIキーは `.env` で管理します。

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxx
```

### Step 2: Claudeクライアントの初期化

```python
import os
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()

client = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))
MODEL = "claude-opus-4-5"
```

### Step 3: アウトライン生成

トピックを受け取り、記事の骨格となるアウトラインをJSON形式で出力させます。JSON形式にしているのは、後続処理でパースしやすくするためです。

```python
def generate_outline(topic: str) -> dict:
    prompt = f"""
あなたは技術ブログの編集者です。
以下のトピックについて、エンジニア向け技術記事のアウトラインをJSON形式で作成してください。

トピック: {topic}

出力形式:
{{
  "title": "記事タイトル",
  "sections": [
    {{
      "heading": "見出し",
      "points": ["カバーする要点1", "カバーする要点2"]
    }}
  ]
}}

制約:
- セクション数は4〜6個
- 実践的でエンジニアが読みたくなる構成にすること
- JSON以外の文字列は出力しないこと
"""

    message = client.messages.create(
        model=MODEL,
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )

    import json
    return json.loads(message.content[0].text)
```

### Step 4: セクションごとの本文生成

アウトラインの各セクションを順番に展開します。前後の文脈をシステムプロンプトで持たせることで、記事全体の一貫性を保ちます。

```python
def generate_section(title: str, heading: str, points: list[str]) -> str:
    points_text = "\n".join(f"- {p}" for p in points)

    prompt = f"""
記事タイトル: {title}
現在の見出し: {heading}
カバーすべき要点:
{points_text}

上記の見出しセクションの本文をMarkdown形式で書いてください。

制約:
- 見出し（##）は含めず、本文のみ出力すること
- 400〜600文字程度
- コード例が必要な場合はコードブロックを使うこと
- エンジニア向けの実践的な内容にすること
- 馴れ馴れしくならず、丁寧だが親しみやすい文体で
"""

    message = client.messages.create(
        model=MODEL,
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )

    return message.content[0].text
```

### Step 5: 記事の組み立てと出力

生成したパーツを組み合わせて完成記事を構築します。

```python
def generate_article(topic: str, output_path: str = None) -> str:
    print(f"[1/2] アウトラインを生成中...")
    outline = generate_outline(topic)

    title = outline["title"]
    sections = outline["sections"]

    print(f"タイトル: {title}")
    print(f"セクション数: {len(sections)}")

    article_parts = [f"# {title}\n"]

    print(f"[2/2] 本文を生成中...")
    for i, section in enumerate(sections):
        print(f"  セクション {i+1}/{len(sections)}: {section['heading']}")
        body = generate_section(title, section["heading"], section["points"])
        article_parts.append(f"## {section['heading']}\n\n{body}\n")

    article = "\n".join(article_parts)

    if output_path:
        with open(output_path, "w", encoding="utf-8") as f:
            f.write(article)
        print(f"\n記事を保存しました: {output_path}")

    return article


# 実行例
if __name__ == "__main__":
    topic = "FastAPIで作るREST API入門"
    article = generate_article(topic, output_path="output.md")
    print(article[:500])  # 冒頭500文字を確認
```

## ハマったポイント・注意点

### 1. JSONパースエラー

アウトライン生成時、Claudeが「```json」のコードブロックで囲んで返してくることがあります。

```python
import re

def extract_json(text: str) -> dict:
    # コードブロックを除去
    text = re.sub(r"```json\n?", "", text)
    text = re.sub(r"```\n?", "", text)
    return json.loads(text.strip())
```

プロンプトに `JSON以外の文字列は出力しないこと` と明示するだけでは完全に防げないため、パース側でも防御的に処理するのがおすすめです。

### 2. レート制限への対処

セクション数が多いとAPIを短時間に連続呼び出しすることになります。レート制限エラー（`RateLimitError`）が発生した場合に備え、リトライ処理を入れておきましょう。

```python
import time
from anthropic import RateLimitError

def call_with_retry(fn, retries=3, wait=10):
    for attempt in range(retries):
        try:
            return fn()
        except RateLimitError:
            if attempt < retries - 1:
                print(f"レート制限に達しました。{wait}秒後にリトライします...")
                time.sleep(wait)
            else:
                raise
```

### 3. 生成内容の品質管理

自動生成ゆえに、たまに事実と異なる記述（いわゆるハルシネーション）が混入します。完全な自動化は難しく、最終的には人間によるレビューが必要です。

実際の運用では「ドラフト生成 → 人間がレビュー・修正 → 公開」というフローにしています。全自動ではなく、**執筆時間を削減するツール**として位置づけるのが現実的です。

### 4. モデル選択について

今回は `claude-opus-4-5` を使用しましたが、用途によっては `claude-haiku-3-5` でも十分な品質が出ます。コストと品質のトレードオフを考慮して選択してください。

| モデル | 特徴 | 用途 |
|---|---|---|
| claude-opus-4-5 | 高品質・高コスト | 本番記事 |
| claude-sonnet-4-5 | バランス型 | 通常用途 |
| claude-haiku-3-5 | 高速・低コスト | プロトタイプ |

## まとめ・今後の展望

### まとめ

今回実装したポイントを振り返ります。

- **二段階生成（アウトライン→本文）** によって品質と安定性を向上
- **セクション分割**でトークン上限問題を回避
- **防御的なJSONパース処理**でエラーに強い実装

約100行のPythonコードで、トピックを入力するだけで構成の整ったMarkdown記事が生成できるようになりました。

現状のシステムをベースに、以下の拡張を検討しています。
